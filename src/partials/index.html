01 The code

Crash recovery is a few lines.

Your agent is a standard OpenAI Agents SDK agent. @register_agent and @step make it durable, and agent_runner runs it inside the workflow, so a crash mid-run resumes from the last completed step instead of starting over.

from agents import Agent, function_tool
from amber import register_agent, agent_runner, step

# wrap tools and external API calls with @step
@function_tool
@step()
def search_web(query: str) -> str:
    ...

# declare the agent with the openai-agents sdk
research_agent = Agent(
    name="research-assistant",
    instructions="...",
    tools=[search_web],
)

# register the agent as a durable Amber workflow
@register_agent(name="research-assistant")
async def research(topic: str) -> str:
    result = await agent_runner(
        starting_agent=research_agent,
        input=topic,
    )
    return str(result.final_output)

02 Durability

Resume from the last good step.

Without durable execution, a crash discards every completed step and the agent restarts from the beginning. Amber saves each step as it finishes, so recovery re-runs only the step that failed.

03 Observability

The agent and the workflow, in one place.

Existing tools see one half each: a tracer watches the agent, a durable engine watches the workflow. Amber puts both in one inspector, so one view covers the agent trace and the recovery state.
