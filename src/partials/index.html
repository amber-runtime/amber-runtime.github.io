01 The code

The crash-recovery code becomes a few decorators.

On the left is crash recovery written by hand: persist each step's output, detect that the process restarted, skip the steps already done, resume from the last one. Drag the handle right to left and all of it disappears. You mark each operation with @step and the agent with @register_agent. Underneath, DBOS checkpoints each step to Postgres, so a crash resumes from the last one that finished. Amber adds the observability and runs as a worker beside your app, with no separate orchestration service to stand up.

Beforeby hand 
# persist every step, detect the restart, skip what
# finished, resume from the last one that did not.
async def research_agent(topic, run_id):
    state = await db.load(run_id)
    done = state.get("steps", {})

    if "search" not in done:
        sources = await web.search(topic)
        done["search"] = sources
        await db.save(run_id, "search", sources)
    sources = done["search"]

    if "summarize" not in done:
        notes = await llm.summarize(sources)
        done["summarize"] = notes
        await db.save(run_id, "summarize", notes)
    notes = done["summarize"]

    if "write" not in done:
        await db.write(topic, notes)
        done["write"] = True
        await db.save(run_id, "write", True)

    return notes
Afterdecorators 
# the same operations, marked as steps. the manual persistence is gone.
from amber import register_agent, step

@step
async def search(topic): ...
@step
async def summarize(sources): ...
@step
async def write(topic, notes): ...

@register_agent(name="research-agent")
async def research_agent(topic):
    sources = await search(topic)
    notes = await summarize(sources)
    await write(topic, notes)
    return notes

03 Observability

The agent and the workflow, in one place.

Existing tools see one half each: a tracer watches the agent, a durable engine watches the workflow. Amber puts both in one inspector, so you can read the agent trace and the recovery state side by side, the thing neither half can show on its own.
