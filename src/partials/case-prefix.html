00 Introduction

Amber is an open‑source, self‑hosted durable execution runtime for long‑running AI agents.

This runtime lets an agent resume precisely from any completed step when failures occur in production.

Amber Runtime · Engineering case study · DBOS + POSTGRES

Durable execution is an approach to executing code in a crash-proof way. It's about ensuring an application can run from start to finish and recover from any failures without starting over. Without a durable execution runtime, developers often have to hand-write and maintain their own fault-handling logic.

AI agents are a rapidly evolving type of application currently. Developers face a tradeoff between agent autonomy versus reliability. AI agents capable of taking real-world actions must touch existing production systems which are often highly distributed and inherently unreliable. These agents will inherit the same risks of failure that warrant durable execution.

In this case study, we introduce how durable execution applies to AI agents, describe how existing durable execution solutions work, and discuss how we addressed agent-specific challenges while building Amber.
